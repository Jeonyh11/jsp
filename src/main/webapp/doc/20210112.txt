
사용자 상태정보 조회 페이지 개발

흐름 
사용자 페이지 화면에서 사용자 정보를 감싸는 tr 태그를 클릭시 해당 사용자의 상세 조회 페이지로 이동


필요한 것들
1.  tr 태그에 이벤트 등록(클릭)
	tr 태그를 클릭시 해당 사용자가 누구인지 판단할 수 있는 기술이 필요
 	요청을 어떻게 보낼 것인가 : javascript
 
2.  kr.or.ddit.user.controller.UserController : 사용자 상세 정보 조회 요청 처리
	1) userid parameter 확인
	2) userService 객체를 이용하여 파라미터에 해당하는 사용자 정보 조회
	3) request 객체에 2)번에 조회된 값을 user라는 속성으로 저장
	4) webapp/user/user.jsp로 화면 생성 위임
	
3.  userForm.html을 이용하여 user.jsp를 만들고 request scope에 담긴 user 속성을 이용하여 사용자 정보 출력
	8가지 정보 출력
	사용자 아이디
	사용자 이름
	비밀번호 : 마스킹 ( ******* )
	등록일시
	별명
	도로주소
	상세주소
	우편번호 코드
	

	
사용자 수정 기능

흐름
사용자 상세조회 페이지 하단에 있는 사용자 수정 버튼을 클릭하면 해당 사용자의 정보를 수정할 수 있는 수정페이지로 이동
( 조회 페이지 구성과 유사 . 조회 페이지의 텍스트 ==> 수정 페이지의 input태그로 변경)

1. 파라미터 전송
	1) 사용자 정보를 조회할 수 있는 최소의 값 : userid (userService를 이용하는 방법) ***
	2) 8개의 사용자 정보를 전부 파라미터로 전송 (userService를 이용하지 않는 방법)

2. kr.or.ddit.user.controller.ModifyUser.java
	userid 파라미터 확인
	userService를 이용하여 사용자 정보 조회
	request객체에 user속성으로 설정
	webapp/user/userModify.jsp로 forward
